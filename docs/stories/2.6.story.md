# Story 2.6 — Feed de Editais

> **Epic:** 2 — Radar de Editais
> **Prioridade:** P0
> **Estimativa:** 1-2 dias
> **Agente:** @dev
> **Depende de:** 2.5, 2.4

---

## Descricao

Tela principal do Radar: lista de editais encontrados, com filtros, busca, ordenacao e status visual.

## Acceptance Criteria

- [ ] Lista de editais em cards ou tabela (toggle de visualizacao)
- [ ] Cada edital mostra: orgao, objeto (truncado), valor estimado, data sessao, portal, status
- [ ] Status visual com badges coloridas: novo (azul), analisando (amarelo), GO (verde), NO-GO (vermelho), etc.
- [ ] Filtros rapidos: por status, portal, data, valor
- [ ] Busca full-text no objeto
- [ ] Ordenacao: mais recente, data sessao, valor
- [ ] Paginacao ou infinite scroll
- [ ] Acao rapida: "Analisar" → vai pra GO/NO-GO
- [ ] Acao rapida: "Descartar" → muda status pra descartado
- [ ] Contador: "X editais novos hoje"
- [ ] Empty state: "Nenhum edital encontrado. Configure seus filtros."
- [ ] Realtime: editais novos aparecem sem reload (Supabase Realtime)
