# Story 1.4 — Auth (Cadastro, Login, Magic Link, Google)

> **Epic:** 1 — Fundacao
> **Prioridade:** P0
> **Estimativa:** 1 dia
> **Agente:** @dev
> **Depende de:** 1.2

---

## Descricao

Implementar autenticacao completa com Supabase Auth: cadastro com criacao automatica de tenant, login por email/senha, magic link e Google OAuth.

## Acceptance Criteria

- [x] Tela de cadastro: nome, email, senha, CNPJ, razao social
- [x] Ao cadastrar, cria automaticamente: user → tenant → profile → subscription (free)
- [x] Login por email + senha
- [x] Login por magic link (link no email)
- [x] Login por Google OAuth
- [x] Tela de "esqueci minha senha" com reset por email
- [x] Middleware de protecao de rotas (redirect pra login se nao autenticado)
- [x] Hook `useAuth()` com user, tenant, profile, loading, signOut
- [x] Supabase trigger: ao criar user, cria profile automaticamente
- [x] Logout funcional
- [x] Redirect apos login: se primeiro acesso → onboarding, senao → dashboard

## Fluxo de Cadastro

```
1. Usuario preenche: nome, email, senha, CNPJ, razao social
2. Supabase cria auth.user
3. Trigger cria profile (vinculado ao user)
4. API cria tenant (com CNPJ)
5. API cria subscription (plano free, trial 14 dias)
6. Redirect → Onboarding wizard (Story 8.1)
```

## Telas

| Tela | Rota | Descricao |
|---|---|---|
| Cadastro | `/auth/signup` | Form de registro |
| Login | `/auth/login` | Email+senha, magic link, Google |
| Reset senha | `/auth/reset` | Envio de email de reset |
| Nova senha | `/auth/new-password` | Form de nova senha (via link) |

## Notas Tecnicas

- Supabase Auth com PKCE flow (mais seguro)
- Middleware Next.js (`middleware.ts`) pra proteger rotas
- Google OAuth: configurar no Supabase Dashboard
- Magic link: template de email customizado
- Rate limiting no cadastro (Supabase ja tem built-in)
