# Story 1.1 — Setup Monorepo Turborepo

> **Epic:** 1 — Fundacao
> **Prioridade:** P0
> **Estimativa:** 1 dia
> **Agente:** @dev

## Descricao

Criar a estrutura monorepo usando Turborepo com os workspaces: web (Next.js), mobile (Expo/React Native) e packages compartilhados.

## Acceptance Criteria

- [x] Monorepo Turborepo inicializado com `pnpm` como package manager
- [x] Workspace `apps/web` com Next.js 15 (App Router, TypeScript)
- [ ] Workspace `apps/mobile` com Expo (React Native, TypeScript) — placeholder criado, setup completo na V2
- [x] Workspace `packages/ui` — componentes compartilhados (shadcn/ui)
- [x] Workspace `packages/db` — cliente Supabase + tipos
- [x] Workspace `packages/shared` — tipos, constantes, utils compartilhados
- [x] Workspace `packages/config-ts` — tsconfig base compartilhado
- [x] Tailwind CSS configurado no web
- [ ] NativeWind configurado no mobile — V2
- [x] `turbo.json` com pipelines: build, dev, lint, typecheck, test
- [x] `.env.example` com variaveis necessarias
- [x] `pnpm dev` roda web (testado, HTTP 200)
- [x] `.gitignore` adequado (node_modules, .env, .turbo, etc.)

## Dependencias Principais

- next@15, react@19, react-dom@19
- tailwindcss, @tailwindcss/postcss
- shadcn/ui (init)

**Mobile (apps/mobile):**
- expo, expo-router
- react-native
- nativewind

**Packages:**
- typescript
- @supabase/supabase-js
- zod

**Dev:**
- turborepo
- eslint, prettier
- vitest

## Notas Tecnicas

- Usar `pnpm` (nao npm ou yarn) — melhor com monorepos
- Next.js com App Router (nao Pages Router)
- Expo com Expo Router (file-based routing)
- shadcn/ui: inicializar com `npx shadcn@latest init`
- Nao instalar ORM ainda (Drizzle vem na Story 1.2)
