## Descricao

Configurar infraestrutura de testes (unit, integration, e2e) e pipeline de CI no GitHub Actions.

## Acceptance Criteria

- [x] Vitest configurado pra unit tests (packages + apps)
- [x] Playwright configurado pra e2e tests (web)
- [x] GitHub Actions workflow: lint → typecheck → test → build
- [ ] Pre-commit hook com lint-staged (lint + format) — pendente (husky requer git init no subdir)
- [x] Coverage report configurado (v8)
- [x] Testes de exemplo funcionando (6 unit, 4 e2e specs)

## Configuracao

**Vitest:**
- Workspace config pra rodar tests de todos os packages
- Coverage com v8
- Path aliases configurados

**Playwright:**
- Testes e2e no `apps/web`
- Base URL configuravel via env
- Screenshot on failure

**GitHub Actions:**
```yaml
on: [push, pull_request]
jobs:
  ci:
    - pnpm install
    - pnpm lint
    - pnpm typecheck
    - pnpm test
    - pnpm build
```
