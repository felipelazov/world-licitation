# Story 2.4 — Worker de Scraping (BullMQ + Redis + Cron)

> **Epic:** 2 — Radar de Editais
> **Prioridade:** P0
> **Estimativa:** 1-2 dias
> **Agente:** @dev
> **Depende de:** 2.2, 2.3

---

## Descricao

Infraestrutura de workers: jobs agendados por cron que executam os scrapers, processam resultados e disparam notificacoes de editais novos.

## Acceptance Criteria

- [x] BullMQ configurado com Redis
- [x] Queue `scraping` com jobs por portal (compras-gov, bec-sp)
- [x] Cron configuravel: 1h (compras-gov), 4h (bec-sp), diario (doc expiry)
- [x] Worker processa job: executa scraper → retorna stats
- [ ] Dashboard de monitoramento (BullMQ Board) — V2
- [x] Retry automatico: 3 tentativas com backoff exponencial
- [x] Dead letter queue pra jobs que falham (removeOnFail: 7 dias)
- [x] Health check endpoint pra workers (/health)
- [x] Logs estruturados (JSON)
- [x] Dockerfile pra deploy
