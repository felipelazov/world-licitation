# Phase: Identidade

**Squad:** squad-design
**Chief:** design-chief
**Agent(s):** marty-neumeier (brand strategy), aaron-draplin (logo), brad-frost (design system)
**Peso:** 15%
**Deliverables:** `deliverables/identidade/`

---

## Objetivo

Criar a identidade visual completa da marca -- estrategia de brand, logo, design tokens e component library -- traduzindo a visao de produto e o posicionamento em uma linguagem visual consistente e memoravel.

## Perguntas-Chave

1. Qual e a personalidade da marca (Neumeier)?
2. Qual e o tom visual -- bold, minimal, premium, playful?
3. Quais cores e tipografia comunicam o posicionamento?
4. Qual e o conceito do logo (Draplin)?
5. Quais componentes de UI sao necessarios para o produto (Frost)?
6. O design system escala para multiplas plataformas?
7. A identidade reflete o WHY e a visao de produto?

## Decisoes Esperadas

1. `brand_personality` -- 3-5 adjetivos que definem a marca
2. `color_palette` -- cores primarias, secundarias e neutras
3. `typography_stack` -- fontes para headings, body e UI
4. `logo_direction` -- conceito aprovado para o logo
5. `component_scope` -- lista de componentes do design system

## Outputs / Deliverables

- `deliverables/identidade/brand-strategy.md` -- estrategia de marca documentada
- `deliverables/identidade/logo-kit/` -- logo em variantes (SVG, PNG, dark, light)
- `deliverables/identidade/design-tokens.yaml` -- tokens de design (cores, espacamento, tipografia)
- `deliverables/identidade/component-library/` -- biblioteca de componentes base

## Criterios de Conclusao

- [ ] Brand strategy documentada com personalidade e tom
- [ ] Logo kit com pelo menos 3 variantes
- [ ] Design tokens exportados em formato consumivel
- [ ] Component library com componentes essenciais definidos
- [ ] Decisoes registradas em `project-state.yaml`

## Proxima Fase

**conteudo** -- A identidade visual alimenta a estrategia de conteudo: cores, tom e componentes definem como o conteudo sera apresentado em cada plataforma.
