## 1. Budget (Orcamento)

- [ ] Prospect confirmou que tem orcamento disponivel para esta solucao
- [ ] Faixa de investimento discutida e compativel com nossa proposta
- [ ] Processo de aprovacao de orcamento mapeado (quem aprova, limites)
- [ ] Fonte do orcamento identificada (verba existente vs nova alocacao)
- [ ] Prospect entende o custo de NAO resolver o problema (custo da inacao)

## 2. Authority (Autoridade)

- [ ] Decisor final identificado por nome e cargo
- [ ] Decisor participou de pelo menos 1 reuniao ou foi mencionado diretamente
- [ ] Influenciadores mapeados (tecnico, financeiro, usuario final)
- [ ] Processo de decisao documentado (comite, individual, consenso)
- [ ] Potenciais bloqueadores identificados e estrategia para cada um

## 3. Need (Necessidade)

- [ ] Dor principal articulada pelo PROSPECT (nao por nos)
- [ ] Impacto do problema quantificado (receita perdida, tempo gasto, risco)
- [ ] Situacao atual mapeada (como resolvem hoje, gaps)
- [ ] Prospect reconhece que precisa de uma solucao externa
- [ ] Necessidade e urgente (nao apenas "seria bom ter")

## 4. Timeline (Prazo)

- [ ] Data desejada de implementacao definida pelo prospect
- [ ] Evento catalisador identificado (deadline, regulacao, meta trimestral)
- [ ] Prazo e realista para nosso processo de entrega
- [ ] Proximos passos com datas acordados entre as partes
- [ ] Risco de adiamento avaliado (probabilidade de "agora nao")

## 5. MEDDIC Complementar

- [ ] Metricas de sucesso definidas (como prospect medira ROI)
- [ ] Economic Buyer engajado (quem assina o cheque)
- [ ] Criterios de decisao documentados (preco, features, prazo, confianca)
- [ ] Processo de decisao com etapas e datas mapeado
- [ ] Dor identificada e conectada a metricas de negocio
- [ ] Champion interno identificado (quem defende internamente)
