## 1. Competitive Alternatives

- [ ] Listamos TODAS as alternativas reais (nao so concorrentes diretos)
- [ ] Incluimos "nao fazer nada" e solucoes manuais (planilha, estagiario)
- [ ] Cada alternativa foi validada com clientes ("o que voce faria se nao existissemos?")
- [ ] Limitacoes de cada alternativa documentadas

## 2. Unique Attributes

- [ ] Cada attribute e REAL e verificavel (nao aspiracional)
- [ ] Nenhuma alternativa tem o mesmo attribute
- [ ] Attributes sao diferenciais tecnicos/funcionais, nao marketing fluff
- [ ] Listamos apenas attributes que importam para best-fit customers

## 3. Value (Attribute → Benefit)

- [ ] Cada attribute foi traduzido em valor para o CLIENTE
- [ ] Value statements usam linguagem do cliente (nao jargao interno)
- [ ] Value themes agrupam attributes relacionados
- [ ] Cada value theme passa o teste "e dai?" — beneficio e concreto

## 4. Best-Fit Customers

- [ ] Best-fit customers sao baseados em dados (nao idealizacao)
- [ ] Temos pelo menos 5 exemplos reais de best-fit customers
- [ ] Sabemos POR QUE eles sao best-fit (o que os diferencia)
- [ ] Nao estamos posicionando para "todo mundo"

## 5. Market Category

- [ ] Categoria faz o valor ser OBVIO sem explicacao
- [ ] Testamos com 5+ clientes: "quando digo [categoria], o que espera?"
- [ ] Expectativa do cliente = nosso valor (alinhamento)
- [ ] NAO escolhemos categoria aspiracional
- [ ] Tipo de categoria definido: existente / subcategoria / nova

## 6. Canvas Completo

- [ ] Os 5 componentes estao documentados no positioning canvas
- [ ] Canvas e compreensivel por alguem de fora do time
- [ ] Positioning statement compilado e claro
- [ ] Canvas alinhado com product vision (Cagan validou)

## 7. Validacao Externa

- [ ] Testado com best-fit customers (minimo 5 conversas)
- [ ] Win rate melhorou apos novo positioning (se repositioning)
- [ ] Squad-sales confirma que pitch baseado no canvas converte
- [ ] Nenhum feedback de confusao sobre o que o produto faz
