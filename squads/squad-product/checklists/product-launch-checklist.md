# Product Launch Checklist

**Produto/Feature:** _______________
**Data de lancamento:** _______________
**Responsavel:** Marty Cagan (Chief) + April Dunford (Positioning)

---

## Pre-Launch: Produto

- [ ] Product vision alinhada com lancamento
- [ ] Discovery completo — feature validada com evidencias
- [ ] Acceptance criteria testados e aprovados
- [ ] Performance dentro dos limites aceitaveis
- [ ] Seguranca revisada (OWASP basics)
- [ ] Rollback plan definido

## Pre-Launch: Positioning

- [ ] Positioning canvas completo e validado (5 componentes)
- [ ] Category design definida
- [ ] Sales pitch estruturado e testado
- [ ] Key messages definidas (3-5 com evidence)
- [ ] Messaging toolkit completo (one-liner, boilerplate, FAQ, don'ts)
- [ ] Testado com best-fit customers (minimo 5)

## Pre-Launch: Retention

- [ ] Hook model projetado para nova feature
- [ ] Engagement metrics definidas com thresholds
- [ ] Trigger strategy definida (externo → interno)
- [ ] Variable reward projetada com variabilidade
- [ ] Ethics check (Manipulation Matrix = Facilitator)

## Pre-Launch: Comunicacao

- [ ] squad-marketing notificado (sinapse) — campanhas prontas
- [ ] squad-sales notificado (sinapse) — pitch atualizado
- [ ] squad-customer-success notificado — base existente comunicada
- [ ] squad-content-distillery notificado — narrativa para atomizar
- [ ] Documentacao/help atualizada

## Lancamento

- [ ] Feature deployed/released
- [ ] Metricas de tracking ativas (analytics)
- [ ] Alertas de engagement configurados
- [ ] Time interno notificado

## Pos-Launch (Semana 1)

- [ ] Metricas D1 coletadas e avaliadas
- [ ] Feedback inicial analisado
- [ ] Bugs criticos triados e resolvidos
- [ ] Ajustes de positioning se necessario

## Pos-Launch (Semana 4)

- [ ] Metricas D7 e D30 avaliadas
- [ ] Hook model validado vs real usage
- [ ] Product review realizado com resultados
- [ ] Decisao: escalar / iterar / pivotar
