# Onboarding Checklist

**Purpose:** Validar que cada etapa do onboarding de cliente foi completada antes de avancar. Cobre o fluxo completo: Welcome → Setup → First Value → Handoff.
**Pattern:** CS-ON (Onboarding Patterns)
**Quality Gate:** BLOCKING — cliente nao avanca de etapa sem checklist da etapa anterior completo.
**Owner:** donna-weber (Onboarding Specialist)

---

## Fase 1: Pre-Onboarding (antes do welcome)

### Recebimento do Cliente de Vendas

- [ ] Dados completos do cliente recebidos (nome, empresa, email, telefone, WhatsApp)
- [ ] Proposta comercial assinada/aceita
- [ ] Plano contratado identificado com funcionalidades incluidas
- [ ] Contexto de venda documentado (dor do cliente, expectativas, promessas feitas)
- [ ] Objecoes superadas durante a venda documentadas
- [ ] Decision maker identificado (quem decide, quem usa)
- [ ] Canal de comunicacao preferido do cliente confirmado
- [ ] Data de fechamento registrada no CRM

**Gate:** Todos os 8 itens devem estar completos para iniciar o welcome.

---

## Fase 2: Welcome (task welcome-client)

### Preparacao

- [ ] Template welcome-package preenchido com dados do cliente
- [ ] Mensagem personalizada (referencia a dor do cliente, nao generica)
- [ ] Proximos passos listados com datas especificas
- [ ] Materiais iniciais preparados (guia, FAQ, video)
- [ ] Call de setup agendada com data e horario

### Execucao

- [ ] Email de boas-vindas enviado
- [ ] Mensagem WhatsApp de confirmacao enviada
- [ ] Cliente confirmou recebimento do welcome
- [ ] Expectativas alinhadas (escopo IN e OUT)
- [ ] Canais de comunicacao confirmados
- [ ] Registro de onboarding criado no CRM

### Verificacao

- [ ] Welcome enviado em ate 24h apos fechamento da venda
- [ ] Cliente respondeu/confirmou em ate 48h
- [ ] Call de setup com data definida (maximo 5 dias uteis apos welcome)
- [ ] Nenhuma promessa feita que nao possa ser cumprida

**Gate:** Welcome completo para iniciar Setup. Minimo: email enviado + call agendada + registro no CRM.

---

## Fase 3: Setup (task setup-account)

### Pre-Call

- [ ] Ambiente de configuracao preparado (credenciais, acessos admin)
- [ ] Materiais de apoio separados para o plano do cliente
- [ ] Roteiro da call preparado (agenda + tempo por topico)
- [ ] Link da call testado e confirmado
- [ ] Lembrete enviado ao cliente 24h antes
- [ ] Lembrete enviado ao cliente 1h antes

### Durante a Call

- [ ] Gravacao iniciada (com consentimento)
- [ ] Agenda compartilhada com o cliente
- [ ] Configuracao feita JUNTO com o cliente (nao por ele)
- [ ] Cada funcionalidade configurada foi testada na hora
- [ ] Duvidas respondidas durante a call
- [ ] Cliente conseguiu realizar acoes basicas sozinho

### Pos-Call

- [ ] Gravacao da call salva e enviada ao cliente
- [ ] Guia de uso rapido enviado
- [ ] FAQ enviado
- [ ] Video tutorial complementar enviado (se aplicavel)
- [ ] Contato de suporte compartilhado
- [ ] Cliente confirmou que consegue acessar e usar sozinho

### Verificacao

- [ ] Conta ativa e funcionando
- [ ] Todas as funcionalidades do plano configuradas
- [ ] Cliente fez login com sucesso apos a call
- [ ] Checklist de configuracao 100% completo no CRM
- [ ] Meta de first value definida com o cliente (o que, quando)

**Gate:** Setup completo para iniciar First Value. Minimo: conta ativa + acesso verificado + meta de first value definida.

---

## Fase 4: First Value (task first-value)

### Planejamento

- [ ] Quick win identificado (resultado rapido e mensuravel)
- [ ] Quick win conectado a dor que motivou a compra
- [ ] Quick win alcancavel em 3-7 dias
- [ ] Baseline coletado (como estava ANTES)
- [ ] Plano de quick win compartilhado com o cliente

### Acompanhamento

- [ ] Check-ins realizados conforme planejado (diarios ou a cada 2 dias)
- [ ] Bloqueios removidos conforme surgiram
- [ ] Progresso documentado em cada check-in
- [ ] Cliente engajado e participando ativamente

### Resultado

- [ ] Primeira vitoria alcancada com evidencia concreta
- [ ] Metricas antes vs depois documentadas
- [ ] Resultado descrito em linguagem do cliente
- [ ] Data de alcance registrada
- [ ] Time to first value calculado (dias desde fechamento)

### Feedback

- [ ] Cliente perguntado: "O que mais te surpreendeu?"
- [ ] Cliente perguntado: "O que poderia ter sido melhor?"
- [ ] Cliente perguntado: "Recomendaria? Por que?"
- [ ] NPS informal coletado (0-10)
- [ ] Permissao para case study solicitada
- [ ] Sinais de interesse em mais funcionalidades registrados

### Verificacao

- [ ] First value alcancado em ate 14 dias apos fechamento
- [ ] Evidencia concreta registrada (screenshot, numero, depoimento)
- [ ] NPS >= 7
- [ ] Feedback completo no CRM

**Gate:** First Value completo para iniciar Handoff. Minimo: vitoria com evidencia + feedback coletado + NPS registrado.

---

## Fase 5: Handoff (task handoff)

### Preparacao do Handoff

- [ ] Documento de handoff completo (9 campos obrigatorios):
  - [ ] Dados do cliente
  - [ ] Contexto de venda
  - [ ] Jornada de onboarding (timeline)
  - [ ] Configuracoes realizadas
  - [ ] First value (resultado e metricas)
  - [ ] Preferencias de comunicacao
  - [ ] Riscos e pontos de atencao
  - [ ] Oportunidades identificadas
  - [ ] Recomendacoes para CS
- [ ] CS responsavel designado pelo CS Head

### Transferencia de Conhecimento

- [ ] Briefing realizado com CS designado
- [ ] CS sem duvidas sobre o cliente
- [ ] Nuances e impressoes pessoais compartilhadas
- [ ] Proximos passos alinhados entre Onboarding e CS

### Apresentacao ao Cliente

- [ ] Call de apresentacao realizada (Onboarding + CS + Cliente)
- [ ] CS demonstrou conhecimento do contexto do cliente
- [ ] Cliente confortavel com a transicao
- [ ] Proxima interacao CS-Cliente definida (data e pauta)

### Formalizacao

- [ ] Owner da conta atualizado no CRM (Onboarding → CS)
- [ ] Onboarding marcado como completo no sistema
- [ ] Email de encerramento de onboarding enviado ao cliente
- [ ] CS Head notificado sobre conclusao
- [ ] Metricas finais de onboarding registradas

### Verificacao

- [ ] Handoff feito via call (nao apenas email)
- [ ] CS briefado ANTES de conhecer o cliente
- [ ] Cliente confirmou conforto com a transicao
- [ ] Zero informacoes perdidas na transferencia

**Gate:** Handoff completo. Onboarding encerrado. CS assume a conta.

---

## Metricas de Qualidade do Onboarding

| Metrica | Target | Como Medir |
|---------|--------|-----------|
| Time to Welcome | <= 24h | Data fechamento → Data welcome enviado |
| Time to Setup | <= 5 dias uteis | Data welcome → Data call setup |
| Time to First Value | <= 14 dias | Data fechamento → Data first value |
| Time to Handoff | <= 21 dias | Data fechamento → Data handoff completo |
| NPS no Onboarding | >= 8 | NPS coletado na fase feedback |
| Activation Rate | >= 90% | Clientes que completaram onboarding / Total |
| Setup Completion Rate | 100% | Funcionalidades configuradas / Total disponivel |

---

## Sinais de Problema no Onboarding

| Sinal | Fase | Acao Imediata |
|-------|------|---------------|
| Cliente nao responde em 48h | Welcome | Follow-up WhatsApp + email |
| No-show na call de setup | Setup | Reagendar em 48h. 2 no-shows = escalar CS Head |
| Quick win nao alcancado em 7 dias | First Value | Pivotar para quick win mais simples |
| Cliente desengajado nos check-ins | First Value | Contato alternativo. Se persistir, escalar |
| Cliente resiste ao handoff | Handoff | Transicao gradual (1-2 calls com CS + Onboarding) |
