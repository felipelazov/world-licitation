# Checklist: Product Vision — Validacao de Visao de Produto

**Agente:** Steve Jobs
**Fase:** Visao de Produto & Foco
**Standard:** Simplicity-First + 1000 No's + Tech x Liberal Arts
**Quando usar:** Apos sessao de visao de produto com Jobs, antes de registrar decisoes

---

## 1. Foco Radical

- [ ] Lista de features essenciais tem NO MAXIMO 3 itens
- [ ] Cada feature essencial justifica sua existencia com impacto direto
- [ ] Cut list documentada com RAZAO para cada corte
- [ ] Nenhuma feature foi mantida "por via das duvidas"
- [ ] Aplica o principio "1000 No's" — disse nao para tudo que nao e essencial

> **Risco se incompleto:** Produto inchado. Feature creep. Faz tudo e nao faz nada bem.

## 2. Experiencia do Usuario

- [ ] Jornada do usuario mapeada em 5-7 passos (inicio ao fim)
- [ ] Cada passo e intuitivo — usuario nao precisa de manual
- [ ] Primeiro contato gera "momento wow" nos primeiros minutos
- [ ] Pontos de friccao identificados e eliminados
- [ ] Experiencia e MEMORAVEL (usuario conta pra alguem)

> **Risco se incompleto:** Produto funcional mas esquecivel. Sem advocacia organica.

## 3. Simplicidade

- [ ] Produto pode ser descrito em UMA FRASE clara
- [ ] Simplicity statement escrito e testavel (alguem de fora entende?)
- [ ] Complexidade interna esta escondida do usuario
- [ ] Nao ha jargao tecnico na comunicacao do produto
- [ ] Design visual reflete simplicidade (se aplicavel)

> **Risco se incompleto:** "O que esse produto faz?" — se a pergunta existe, ta complexo.

## 4. Excelencia e Padrao

- [ ] Cada feature essencial e EXCELENTE, nao apenas funcional
- [ ] Padrao de qualidade definido para cada entregavel
- [ ] Nenhum componente e "ok por enquanto" — tudo e intencional
- [ ] Referencia de excelencia identificada (benchmark)
- [ ] Fundador se orgulha de mostrar o produto como esta

> **Risco se incompleto:** Produto mediocre. "E bom, mas..." — o "mas" mata.

## 5. Alinhamento Estrategico

- [ ] Visao de produto conecta com WHY (Sinek)
- [ ] Features essenciais entregam o Dream Outcome (Hormozi)
- [ ] Cut list nao contradiz a Value Stack da oferta
- [ ] Decisoes prontas para sinapse → squad-design (o que construir)
- [ ] Decisoes prontas para sinapse → squad-content-distillery (como comunicar)

> **Risco se incompleto:** Produto desconectado da estrategia. Cada squad puxa pra um lado.

## 6. Coragem nas Decisoes

- [ ] Pelo menos 1 decisao e "controversa" (cortar algo que parece importante)
- [ ] Nenhuma decisao foi tomada por medo ou pressao de mercado
- [ ] Decisoes refletem convicção, nao consenso
- [ ] Fundador DEFENDE as escolhas com argumentos claros

> **Risco se incompleto:** Produto safe. Nao ofende ninguem, nao inspira ninguem.
