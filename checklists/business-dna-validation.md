# Checklist de Validacao — Business DNA Blueprint

Executar APOS a conclusao da fase business-dna.
Garante que o Business DNA Blueprint esta completo, rastreavel e pronto para alimentar squads downstream.
Sem isso, squads recebem briefs incompletos ou desalinhados com as decisoes dos advisors.

## 1. Proposito & Narrativa Estrategica

- [ ] WHY/HOW/WHAT traduzidos em storytelling coeso (nao bullets)
- [ ] Narrativa rastreavel as decisoes do Simon Sinek registradas no state
- [ ] Causa e posicionamento claros para qualquer leitor externo
- [ ] Conexao explicita entre genius profile e narrativa (se descoberta foi feita)
- [ ] Tom e linguagem consistentes com o publico-alvo definido

> **Risco se falhar:** Squads downstream interpretam o proposito cada um a sua maneira. Branding e conteudo ficam desalinhados.

## 2. Oferta Irrecusavel

- [ ] Grand Slam Offer formatada com dream outcome, likelihood, time delay, effort/sacrifice
- [ ] Preco e garantia documentados
- [ ] Oferta rastreavel as decisoes do Hormozi registradas no state
- [ ] Formato utilizavel por outros squads (nao so notas internas)
- [ ] Diferenciais listados com clareza

> **Risco se falhar:** Squad de vendas cria pitch sem base. Marketing promete o que a oferta nao entrega.

## 3. Produto Essencial

- [ ] Features essenciais listadas com rationale de inclusao
- [ ] Features cortadas listadas com rationale de exclusao
- [ ] Decisoes rastreaveis ao Jobs registradas no state
- [ ] Experiencia do usuario descrita de ponta a ponta
- [ ] Simplicity statement presente (produto em 1 frase)

> **Risco se falhar:** Escopo creep. Features cortadas voltam sem justificativa. Produto perde foco.

## 4. Anti-Briefing

- [ ] >= 5 itens claros do que o negocio NAO e
- [ ] Cada item tem justificativa concreta
- [ ] Anti-briefing cobre posicionamento, produto, publico e tom
- [ ] Itens sao uteis para squads (evitam decisoes erradas)

> **Risco se falhar:** Squads fazem coisas que o negocio nao deveria fazer. Design cria identidade generica. Conteudo fala com publico errado.

## 5. Mapa de Decisoes

- [ ] Todas as decisoes dos 3 advisors mapeadas
- [ ] Cada decisao tem: advisor, descricao, impacto, squads afetados
- [ ] Formato tabela legivel e consultavel
- [ ] Nenhuma decisao significativa omitida
- [ ] Consistencia entre decisoes (sem contradicoes)

> **Risco se falhar:** Decisoes perdidas. Squads contradizem advisors sem saber. Contexto se perde entre sessoes.

## 6. Briefs por Squad

- [ ] Um brief gerado para cada squad ativo no projeto
- [ ] Cada brief contem: contexto, o que fazer, o que NAO fazer, KPIs relevantes
- [ ] Briefs sao acionaveis (nao genericos)
- [ ] Briefs referenciam decisoes especificas do mapa
- [ ] Briefs salvos em `deliverables/business-dna/squad-briefs/`

> **Risco se falhar:** Squads comecam do zero. Toda a inteligencia dos advisors se perde na transicao.

## 7. Scorecard

- [ ] KPIs definidos com metrica clara
- [ ] Cada KPI tem baseline (estado atual) e target 90 dias
- [ ] Squad responsavel por cada KPI identificado
- [ ] KPIs cobrem as dimensoes criticas do negocio
- [ ] Formato consultavel e atualizavel

> **Risco se falhar:** Ninguem sabe o que medir. Sucesso fica subjetivo. Accountability inexistente.

## 8. PRD Bridge (Condicional — apenas se tipo = app ou hibrido)

- [ ] Features mapeadas para Functional Requirements (FR)
- [ ] Metricas mapeadas para Non-Functional Requirements (NFR)
- [ ] Publico-alvo mapeado para user stories
- [ ] Pricing/constraints documentados
- [ ] Documento ponte criado em `deliverables/business-dna/prd-bridge.md`
- [ ] Rastreabilidade: cada FR/NFR aponta para decisao de advisor

> **Risco se falhar:** Time tecnico constroi sem contexto de negocio. Features sem justificativa. PRD desconectado da estrategia.
